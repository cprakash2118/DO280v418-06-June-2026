3.3. Defining and Applying Permissions with RBAC

Rule - Action 
ROle - Set of Action 
ROleBinding - Assign role to user/group.

RBAC Scope:-
- namespace/project level  
- cluster level 
 
 Local RBAC
 $ oc adm policy add-role-to-user/group roleName user/group -n namespaceName
 $ oc adm policy remove-role-from-user/group roleName user/group -n namespaceName
 
 Cluster RBAC:-
 $ oc adm policy add-cluster-role-to-user/group roleName user/group 
 $ oc adm policy remove-cluster-role-from-user/group roleName user/group
 
 
Default Roles:-
Local RABC (w.r.t namespace) 
 - admin - Admin access to namespace (add new admin, provides edit/view access w.r.t namespace)
 - edit  - Edit access (deploy application - create all k8s/ocp required resources)
 - view  - ONly read access 
  
Cluster RBAC:-
 - cluster-admin - This is like root 
 - cluster-reader - view access to cluster level 
 - cluster-status  - Only status info 
 - self-provisioner - Allow to create namespace/project cross cluster.
 
 
Users Type:-
 - Normal 
 - serviceaccount 
 - system account 
   System user names start with a system: prefix, 
   Example:
   such as system:admin, system:openshift-registry, and system:node:node1.example.com
   
Group Management:-
$ oc adm groups new lead-developers
$ oc adm groups add-users lead-developers user1
