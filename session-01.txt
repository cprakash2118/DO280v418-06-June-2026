### Point-02:-
Classroom environment (FYI- You consider this like a prod setup and relate).

1-) Users Details:-
    - Login to VMs/Servers
	  student/student [sudo access]
	  root/redhat 
	  
	- Login to Red Hat OpenShift cluster.
      admin/redhatocp [it's cluster-admin access ; which is same as root access w.r.t Linux OS]
      developer/developer [basic user]

    - Login to OCP cluster, few required resources.
      API URL:- 
	     - https://api.clusterName.baseDomain:6443 
		   Example:
		   - https://api.ocp4.example.com:6443 
		   
	$ oc login -u admin -p redhatocp https://api.ocp4.example.com:6443  	   

	  Web URL:- As per LAB (DO280).
       https://console-openshift-console.apps.ocp4.example.com 	(admin/redhatocp).

2-) Domain & Networks:-
    
	Network Name              Domain Name                IP Range 
    Classroom Network         example.com                172.25.252.0/24 
    Student Network           lab.example.com            172.25.250.0/24 
    Cluster Network           ocp4.example.com           192.168.50.0/24 


3-) VMs/Servers:-
   - workstation.lab.example.com     - Jump server; Connect to OCp cluster via CLI/GUI.
   - bastion.lab.example.com         - Connect Student to Classroom network; It also act as DNS server.
   - classroom.example.com           - Proxy node to connect to Internet (quay.io;github.com etc).
   - utility.lab.example.com         - Main node - Imp services like DHCP, PXE/KickStart, HAproxy, tftp, Apache Service.
   - registry.ocp4.example.com       - Images & code repo also (git.ocp4.exampe.com)
   - idm.ocp4.example.com            - LDAP server.
   - master01.ocp4.example.com       - OCP cluster node (Master/worker)
