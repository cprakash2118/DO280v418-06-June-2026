# Role Based Access Control (RBAC).
-1# Rule:- One action to be taken 
-2# Role:- Set of action 
-3# RoleBinding:-  Assign role to user/group.

# Scope of RBAC w.r.t OpenShift:-
- CLuster level 
- Local/Namespace/project level 
FYI- Without namespace application deployment/anything not possible.

## Default roles:-
Cluster Roles:-
1-# cluster-admin (super user role w.r.t cluster - like root account in Linux)
2-# cluster-reader (can read anything accross cluster.
3-# cluster-status (Just to get the cluster status)
4-# self-provisioner (allow to create project/namespace).

local/namespace/project:-
5-# admin w.r.t namespace
6-# edit w.r.t namespace
7-# view w.r.t namespace
